in mongoDb data is inserted in the formatt of BSON(Binary JSON)
in JSON data is text based which is difficult while parsing and JSON is space inefficient as it is text based
BSON solves the above problems of JSON
we give JSON to MongoDB but it stores as BSON
BSON has more data types than JSON


JSON vs BSON
                                        JSON	                                              BSON       
Encoding                             UTF-8 string                                            Binary
Data Support            String, boolean, number, array, object, null            String, boolean, number (integer, float, long, decimal128...), array, null, date, BinData
Readability                        Human and machine                                        Machine only
 
Documents: Mongo stores data in the form of documents(BSON doc). (like rows in sql)
collections: Mongo stores documents in collections.(like tables in sql)


https://www.mongodb.com/docs/manual/crud/   (to know command to crud operations)

commands :-
 * insert a single doc : db.<collection_name>.insertOne()  (if there is no collection before, it creates new collection)
 //give doc as a js object
 eg :
 db.student.insertOne({name: "saikrishn" , marks: 98});
 to see docs in a collection : db.<collection_name>.find()
 eg:
         db.student.find()
         output: 
        [
        {
            _id: ObjectId('69b2fba360a18a1efe7c2907'),
            name: 'Saikrishn',
            marks: 98
        }
        ]

Note : here in mongoDb's collection the docs can be with different schemas inlike sql has fixed schemas for all tuples of a table

  * to insert many docs at a time : db.<collection_name>,insertMany(<Array of Doc>)
  eg: db.student.insertMany([{name: "ram", marks: 98},{name: "ravi", marks: 88},{name: "charan", marks: 78}])

        output:
        {
            acknowledged: true,

* db.<collection_name>.find({key: value})  (to get all docs those contain the given key value pair)
and it returns cursor means reference to original
